Le répertoire est organisé en plusieurs TD. Chaque TD comprend les dossiers "tst" et "src". "tst" regroupe tous les fichiers liés aux tests et "src" tous les fichiers liés aux sources.

Pour compiler le projet, il y a un makefile différent dans chaque dossier TD. Une règle générale "make" permet de compiler tout le projet.
La commande "make" compile puis lance les exécutables ainsi que les tests.
La commande "make run" permet de lancer l'exécutable.
La commande "make memo" permet de lancer l'exécutable et d'en stocker les données dans un tableau.
La commande "make file" permet de lancer l'exécutable et d'en stocker les données dans le fichier simple.txt.
La commande "make test" permet compiler les tests et de lancer l'exécutable.
La commande "make clean" permet de supprimer tous les fichiers .class des répertoires.
